Git/Github 과제 

Git의 정의
: 버전 관리 시스템
-> 파일 변경사항을 시간에 따라 기록해서 필요시에 그 버전을 불러올수있게함

Git으로 Github에 commit, push 하는 과정
: git add . 혹은 git add [원하는 파일명]
-> git commit -m "커밋 내용"
-> git push [원격 저장소명] [브랜치명]

_________________________________________________________________
Github의 정의
: Git으로 관리하는 프로젝트를 업로드 할 수 있는 사이트
-> Git의 버전관리시스템을 이용하여 협업에 용이하다

주요 기능 : pull request, history
pull request(PR) : 팀원들에게 작성한 코드를 공유하고 의견을 주고받을수있는 공간

history : 누가 언제 어떤 내용을 수정했는지 기록함 
실수,버그가 생겼을때 과거로 돌아갈수있음
